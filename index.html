Kerberoasting em Active Directory e Hardening

Demonstracao em ambiente controlado usando Windows Server como Domain Controller e Kali como maquina de teste.

Ideia central: uma conta comum pode solicitar ticket Kerberos para uma conta de servico com SPN. Se a senha for fraca, o hash pode ser quebrado offline.
Ambiente
Conceito

O que é Kerberoasting?

Analogia: Alice é uma funcionária comum. Ela pede na recepção um crachá para visitar a sala do SQL. O crachá é válido, mas foi protegido com uma chave baseada na senha da conta de serviço.
  • Funcionário comum: alice
  • Recepção: Domain Controller
  • Sala/serviço: MSSQLSvc/sql.lab.local:1433
  • Responsável pela sala: svc_sql
  • Crachá temporário: ticket Kerberos TGS
Conceito
Windows Server

AD DS e DNS no Domain Controller

O Windows Server atua como controlador do domínio lab.local.

AD DS autentica usuários e armazena objetos do domínio. DNS permite localizar serviços e o próprio DC.

Detecção

Event ID 4769

O evento 4769 registra solicitação de ticket Kerberos de serviço, TGS.

Muitos eventos para várias contas de serviço podem indicar atividade suspeita de Kerberoasting.

Monitoramento4769
Conclusão

Como corrigir o risco

  • Remover contas de serviço de grupos privilegiados.
  • Usar senhas longas, aleatórias e únicas.
  • Preferir gMSA quando possível.
  • Revisar SPNs periodicamente.
  • Monitorar eventos Kerberos, principalmente 4769.
Kerberoasting mostra que o risco muitas vezes vem de configuração fraca, não de uma falha nova do Windows.
